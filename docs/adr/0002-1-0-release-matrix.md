# Use a bounded 1.0 release matrix instead of exhaustive scene-template acceptance

The public 1.0 release will require an end-to-end formal-delivery benchmark for each of 开题答辩、中期考核、毕业答辩、组会-文献精读、组会-周报进展、组会-课题进展、科研项目申报、科研项目比赛、项目中期与结题、学术会议报告; grammar/render/identity regression for bundled templates T01 through T08; and selected high-risk scene-template combinations. We reject complete Cartesian-product manual acceptance because it would multiply release cost without proportionally improving confidence, while a representative-only MVP would violate the all-supported-scenes release commitment.
