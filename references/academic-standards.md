# 国内学术答辩 PPT 规范

> 供 Phase 2/3 使用。当用户模板缺乏明确定义时，以下规范作为兜底标准。
> 若用户模板本身有明确规定，以模板为准。

---

## 1. 字体规范

### 原则：模板原生字体优先

默认完全保留用户模板的字体定义。仅在模板字体缺失或无法解析时使用兜底方案。

### 兜底字体方案

| 用途 | 中文 | 英文 |
|---|---|---|
| 页面大标题 | 微软雅黑 (Bold) | Times New Roman (Bold) |
| 小标题/副标题 | 微软雅黑 | Times New Roman |
| 正文 bullet | 微软雅黑 | Times New Roman |
| 图表标注/表格文字 | 微软雅黑 | Times New Roman |
| 页码/页脚 | 微软雅黑 | Times New Roman |

**中英文字体的正确混合**：在 python-pptx 中，每个文本 run 需要分别设置东亚字体（`a:ea`）和拉丁字体（`a:latin`），确保中文用微软雅黑、英文和数字用 Times New Roman 自动切换。

### 代码/等宽文字（如适用）

如果 PPT 中出现代码片段，使用 Consolas 或 Courier New。

---

## 2. 字号规范

### 原则：模板占位符字号优先。仅当模板未定义时参考以下区间：

| 元素 | 字号范围 |
|---|---|
| 页面大标题 | 24–30 pt |
| 小标题/板块标签 | 16–20 pt |
| 正文 bullet 主体 | 14–16 pt |
| 正文 bullet 子项 | 12–14 pt |
| 图表图注 (caption) | 10–12 pt |
| 表格内文字 | 10–12 pt |
| 页码/页脚 | 8–10 pt |
| 封面主标题 | 32–40 pt |
| 封面副标题/信息 | 16–20 pt |
| 致谢/结尾大字 | 40–56 pt |

---

## 3. 配色规范

### 原则

- 学术答辩 PPT 配色应**克制且一致**
- 整份 PPT 使用 1 个主色调 + 1-2 个辅助色 + 中性色（白/灰/黑）
- 避免超过 4 种独立颜色同时出现在同一页

### 推荐主色调方向

| 场景 | 推荐主色方向 | 示例 hex |
|---|---|---|
| 理工科答辩 | 深蓝 / 藏蓝 | `#1B3A5C` `#2B579A` |
| 医学/生物 | 蓝绿 / 深绿 | `#1B5C4A` `#2E7D32` |
| 人文社科 | 深红 / 暗紫 | `#7B2D3B` `#5B2C6E` |
| 通用安全色 | 深蓝灰色 | `#2C3E50` `#34495E` |

### 中性色

| 用途 | 颜色 |
|---|---|
| 页面背景 | 纯白 `#FFFFFF` 或极浅灰 `#F8F9FA` |
| 正文文字 | 深灰 `#2C3E50`（非纯黑，纯黑对比度过高） |
| 次要文字/图注 | 中灰 `#7F8C8D` |
| 分割线 | 浅灰 `#DDDDDD` |
| 强调/高亮 | 暖橙 `#E67E22`（仅少量使用于关键数据或箭头） |

## 4. 版式惯例

### 页面结构

```
┌─────────────────────────────────────┐
│  [装饰线/学校标识]                  │  ← 顶部装饰区（0.5"）
├─────────────────────────────────────┤
│                                     │
│  页面标题（28pt, Bold）              │  ← 标题区
│  副标题/板块标注（14pt, 辅助色）      │
│                                     │
├─────────────────────────────────────┤
│                                     │
│  ● 核心论点一                        │
│    — 支撑数据/说明                   │  ← 正文区
│  ● 核心论点二                        │
│    — 支撑数据/说明                   │
│                                     │
│         [图表区域]                   │
│                                     │
├─────────────────────────────────────┤
│  页码                      学校/院系  │  ← 页脚区（0.4"）
└─────────────────────────────────────┘
```

### 常见页面版式

1. **封面**：题目居中或左对齐 + 答辩人信息 + 学校标识 + 日期
2. **章节过渡页**：大字章节标题 + 本章要点（3-5 条 bullet）+ 区分色块或大号序号
3. **纯文字页**：标题 + bullet 列表（3-6 条），子 bullet 缩进
4. **图文混排（左文右图）**：左侧 40-45% 文字 + 右侧 55-60% 图表
5. **图文混排（上图下文）**：上方 60-70% 图表 + 下方 30-40% 文字说明
6. **多图网格**：2×2 或 2×3 均等网格，每格含图 + 简短标签
7. **双栏对比**：左右均分，适合方法对比、实验结果对比
8. **结尾/致谢**：大字"感谢聆听/请批评指正" + 联系方式（可选）

## 5. 内容密度指南

### 论证密度优先

- 普通内容页默认包含：1 个结论式标题、2-4 个支撑证据单元、1-2 个证据载体、1 个解释/边界和 1 个承接关系。
- 信息密度不以字数直接衡量。图、表、公式、流程图、指标卡、对比关系都属于信息载体。
- 有图有字不是机械要求：多图证据页可以以图为主，定义或局限页可以以文字为主；每个元素必须服务于本页论点。
- 三栏、四栏、五栏模板只有在每栏都有明确语义任务时才能使用。内容不足时应换紧凑原生版式、调整原框尺寸或合并页面。
- 章节页、目录页和结束页允许留白；普通内容页不得因为材料提取不足而保留大面积无意义空框。

### Bullet 规范
- 每页 bullet 主体 3-5 条为佳，最多不超过 6 条
- 每条 bullet 1-2 行为佳，不超过 3 行
- 子 bullet（—开头）不超过 2 层嵌套
- 避免在 bullet 中使用完整句子——用关键词和短语

### 图表密度
- 单页 1-2 张大图为佳（每张足够大，观众能看清）
- 3-4 张小图可用 2×2 网格
- 5+ 张图需要谨慎评估——是否真的需要全部展示？

### 文字密度
- 学术答辩 PPT 不追求"一句话一页"的 TED 风格
- 但也不应该把论文段落直接粘贴到 PPT 上
- 一个好的学术 PPT 页面：观众能在 15-20 秒内抓住核心信息
- 讲述者应能用“结论 -> 证据 -> 解释/边界 -> 下一页”顺畅讲完本页；缺少任一关键环节时优先补充材料或重组页面，而不是添加装饰文字

## 6. 表格在 PPT 中的呈现

### 简化原则

学术 PPT 中的表格应比论文中的表格更精简：

- **列数**：建议 ≤ 5 列（论文中的宽表需要精简或拆分）
- **行数**：建议 ≤ 10 行（超过时考虑只展示代表性数据行）
- **数字精度**：小数点后保留 1-2 位即可（论文中可能是 4 位）
- **强调**：最优结果用粗体或不同颜色标注

### 表格来源

表格直接从源文档提取，不做自动样式转换。如果用户需要 PPT 中的表格与论文格式一致，应在 Phase 1 确认时指明。

---

## 7. 页码与导航

- 除封面外，所有页面建议添加页码（右下角）
- 长答辩（>20 页）建议添加章节导航标记（如顶部或侧边的小色块标识当前章节）
- 页码字号 8-10 pt，不影响正文阅读
