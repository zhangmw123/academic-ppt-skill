# 科研绘图提示词库

> 供 Phase 3 渲染时使用。在需要生成技术路线图、架构图或统计图表时，选择对应
> prompt 模板，填入具体内容后交给当前环境可用的绘图工具或 matplotlib。

## Part 0：先生成结构化绘图任务

提示词只负责辅助理解，不能直接作为绘图执行合同。先在
`visual_tasks.json` 中固定页面观点、证据编号、节点、边、数值来源、输出路径和
降级方案，再选择后端：

- 有数值轴：matplotlib；
- 有框和箭头：PowerPoint 原生形状，Graphviz 可用于计算坐标；
- 有真实植物、设备或场景：优先论文原图，其次检索可授权网络图片；
- 复杂概念性科研插画：使用可用的 academic plotting 或 image generation 工具；
- 所有关键标签、数字和箭头最终保持为可编辑 PowerPoint 元素。

架构图生成前必须输出节点表和边表并校验。AI 生成的架构图不能直接承担精确
文字和关系；它只能作为构图参考或无文字视觉底稿。

## Part A：学术架构图

### 适用场景
- 技术路线图（多阶段流水线）
- 系统架构图（多层架构、模块交互）
- 方法流程图（数据从输入到输出的完整链路）
- 模型结构图（Agent 架构、神经网络组件）

### Prompt 模板

```
你是一位世界顶尖的学术插画专家，专注于为计算机科学领域的顶级会议（如 CVPR, NeurIPS, ACL, ISWC）绘制高质量、直观且美观的论文架构图。

请基于以下方法描述，设计并绘制一张专业的学术架构图。

## 方法描述
[METHOD_DESCRIPTION]

## 视觉约束

### 风格基调
- 顶会论文风格：专业、干净、现代、极简主义
- 核心美学：扁平化矢量插画风格，线条简洁，参考 DeepMind 或 Google Research 论文中的图表美学
- 拒绝卡通感、油画感或过度艺术化，保持严谨的学术图表美学
- 背景必须是纯白色，无任何纹理或阴影

### 色彩体系
- 严格使用淡色系或柔和色调
- 严禁使用过于鲜艳饱和的颜色（如大红大绿）或过于暗淡沉重的颜色
- 利用颜色的深浅变化区分不同的模块类型
- 建议主色调方向：
  - 蓝色系：数据流、模型推理、核心算法
  - 灰色系：输入数据、外部资源、存储
  - 绿色/青色系：输出结果、评估指标
  - 橙色/暖色系：人工干预、标注、反馈

### 内容与布局
- 将方法论转化为清晰的模块和数据流箭头
- 模块间用方向明确的箭头连接，表明数据/控制流方向
- 适当使用现代简洁的矢量图标嵌入模块以增强直观性
- 布局优先采用：从左到右（时间/流程）、从上到下（层次/抽象级别）、或两者结合

### 文字规范
- 标签语言服从整套 PPT 语言；中英文术语沿用论文原文
- 为每个关键模块添加清晰易读的文本标签（简短名词短语，1-4 个词）
- 严禁在图中出现长句子、描述性段落或复杂的数学公式
- 文字的作用是标识模块身份，不是解释原理

### 禁止事项
- 不允许逼真照片感或 3D 渲染效果
- 不允许杂乱的草图线条
- 不允许难以辨认的文本（字号过小或与背景对比度不足）
- 不允许廉价的 3D 阴影或渐变瑕疵

## 输出
优先输出节点/边结构和可编辑 PowerPoint 图形；需要 AI 绘图时输出无关键文字的
PNG 视觉底稿，随后在 PowerPoint 中重建标签、箭头和数字。
```

### 使用方式

将 `[METHOD_DESCRIPTION]` 替换为论文方法描述。优先复用论文原图；确实需要
重绘时，使用当前环境可用的图像/绘图 skill。没有专用绘图 skill 时，使用
PowerPoint 基础形状或 matplotlib 构建可编辑、克制的流程图。不要依赖一个
未安装的固定 skill 名称。

## Part B：实验数据图表（matplotlib 生成）

### 适用场景
- SOTA 性能对比（柱状图/条形图）
- 消融实验（分组柱状图）
- 训练趋势（折线图 + 置信区间）
- 多指标综合评估（雷达图/热力图）
- 错误分析（混淆矩阵/分布图）

### Prompt 模板

```
你是一位就职于顶级科学期刊的资深数据可视化专家。你拥有极高的学术审美，严谨且专业。请分析实验数据，从标准学术图表库中选择最能证明实验有效性的方案，并给出详细的视觉设计规范。

## 实验数据与核心结论
[DATA_AND_CLAIM]

## 标准学术图表库（按优先级排列）

### 一、数值与性能对比类
1. 纵向分组柱状图 — 标准 SOTA 对比，对比项适中、标签较短时首选
2. 横向条形图 — 方法名较长或对比项非常多时推荐，避免 X 轴文字倾斜
3. 帕累托前沿图 — 展示两个相互制约指标的权衡关系
4. 雷达图 — 多维度综合能力评估（速度/精度/显存/鲁棒性等）
5. 堆叠柱状图 — 展示整体指标的细分构成

### 二、趋势与收敛类
6. 带置信区域的折线图 — 训练 Loss/Accuracy 趋势，半透明阴影表示多次实验的标准差
7. 局部放大折线图 — 多个模型后期收敛接近时，嵌入放大子图展示微小优势
8. 散点拟合图 — 离散数据的整体趋势，配合拟合曲线

### 三、模型评估与分类类
9. ROC 曲线 — 二分类，正负样本平衡
10. Precision-Recall 曲线 — 类别不平衡，正样本极少时比 ROC 更真实

### 四、数据关系与矩阵可视化类
11. 热力图 — 大规模矩阵数据（混淆矩阵、多模型多任务对比）
12. 散点图 — 两个连续变量的相关性（预测值 vs 真实值，配合对角参考线）
13. 气泡图 — 散点图 + 第三维度（参数量/计算成本）

### 五、统计分布与构成类
14. 小提琴图 — 展示概率密度分布形状，优于箱线图
15. 箱线图 — 多组数据的分布范围、中位数及离群点
16. 环形图 — 分类数据占比（错误类型分布等），优先环形而非扇形

### 六、复合布局类
17. 双 Y 轴图 — 两个量纲完全不同的变量同图展示（如左轴精度/右轴显存）
18. 柱折组合图 — 柱状图作背景（样本数），折线图作前景（精度），适合长尾分布
19. 分面网格图 — 对比变量过多时，拆分为矩阵排列的小图共享坐标轴

## 约束

### 来源优先
- 优先从上述列表中选择
- 若有更适合的顶会标准图表也可推荐，但杜绝非学术的商业图表

### 统计严谨
- 多次实验数据 → 必须添加误差线或置信区间
- 单次实验数据 → 不强行添加
- 若做了统计检验（如 t-test），标注显著性（* p<0.05, ** p<0.01, *** p<0.001）

### 尺度适应性
若数据组间差异巨大，根据情况选择最佳补救：
- 关注绝对数值 → 断裂坐标轴
- 跨越数量级或指数变化 → 对数坐标
- 关注相对提升幅度 → 归一化

### 视觉逻辑
- 标签长度决定横向/纵向
- 数据维度决定单轴/双轴
- 颜色不超过 5 种，优先用同色系深浅区分

### 中文字体（Windows 环境）
- matplotlib 需显式设置中文字体为 SimHei 或 Microsoft YaHei
- 或在脚本开头添加：
  ```python
  import matplotlib.pyplot as plt
  plt.rcParams['font.sans-serif'] = ['Microsoft YaHei', 'SimHei']
  plt.rcParams['axes.unicode_minus'] = False
  ```

## 输出格式

请严格按以下结构输出：

1. **推荐方案**：图表名称（从上述库中选择 1-2 种）
2. **核心理由**：结合数据逻辑，解释为什么这张图最符合当前的学术叙事需求
3. **视觉设计规范**：
   - 坐标轴：X/Y 轴物理含义及单位
   - 尺度处理：断裂轴/对数坐标/归一化的具体建议（如适用）
   - 统计要素：误差线/拟合曲线/显著性标记（如适用）
   - 配色与样式：具体配色策略及线型建议
4. **生成方式**：matplotlib 脚本的关键参数建议
```

### 使用方式

Claude 在 Phase 3 需要生成实验图表时：
1. 读取 outline.json 中对应页面的 `chart_description` 和绑定的数据

## Part C：外部实物图和背景图

当论文没有可用的植物、材料、设备或场景图时，使用环境可用的图片搜索或网页
搜索工具。检索并不等于授权，必须进入 `visual_tasks.json`：

1. 保存检索词、来源落地页和直接图片地址；
2. 保存作者、许可证、访问日期和本地文件路径；
3. 仅接受 Public Domain、CC0、CC BY、CC BY-SA、用户提供或明确授权；
4. 优先选择主体明确、无遮挡、分辨率足够的图片；植物/器官优先透明背景；
5. 背景图只承担氛围，不承担事实证明，降低对比度并避开正文区；
6. 不重复使用同一图片，除非同一视觉被明确设为跨页母题；
7. 按许可证要求在备注或资产清单中署名。
8. 下载后用 `normalize_image_asset.py` 转为无 EXIF/透明通道的 RGB JPEG 或 PNG，
   再插入 PowerPoint。

找不到许可清晰的图片时，退回论文原图、用户提供图片、可验证的公共领域图库，
或使用 AI 生成的非事实性背景。不得直接下载搜索结果缩略图。

画一张学术风格的架构图/流程图，纯白背景，扁平化矢量风格。

```

### 统计图快速模式
```
用 matplotlib 画一张[图表类型]。
数据：[粘贴数据或描述]
X 轴：[含义]  Y 轴：[含义]
要求：学术配色、添加误差线(如有)、中文字体支持。
输出：PNG, dpi=200。
```

## 附录：常用学术配色参考

### 蓝色主色系（推荐用于架构图）
- 主色 #2B6CB0（深蓝，核心模块）
- 辅色 #4299E1（中蓝，次要模块）
- 浅色 #BEE3F8（极浅蓝，背景/容器）
- 强调色 #F6AD55（暖橙，仅用于关键箭头或高亮）

### 多组对比色（推荐用于统计图的 5 组以内对比）
- #2B6CB0（蓝）— 本论文方法
- #A0AEC0（灰）— Baseline/传统方法
- #68D391（绿）— 次优方法
- #F6AD55（橙）— 参考线/阈值
- #F687B3（粉）— 辅助数据

### 热力图配色
- 推荐 cmap='Blues' 或 cmap='YlOrBr'（浅→深，直观）
- 避免 jet 或 rainbow colormap（已被学术圈广泛批评）
