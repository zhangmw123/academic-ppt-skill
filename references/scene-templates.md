# 第一阶段科研汇报场景配置

本文件供内容规划使用；机器字段以 `scene-profiles.json` 为准。

## 总则

区分两类结构：

- `display_sections`：用户在目录页看到的一级板块，可以选择预设，也可以替换、合并、拆分或完全自定义；
- `required_argument_units`：该场景必须讲清楚的论证环节。修改目录后仍需检查这些环节是否被其他板块覆盖。

不要按论文目录机械拆页。先回答场景要证明什么，再把材料组织成可讲述的链条。

## 开题答辩

目标：证明选题值得做、空白真实、目标明确、方案可行、已有基础且能够按期完成。

可选分块：

1. 标准六板块：研究背景与意义 / 国内外研究现状 / 研究目标与内容 / 研究方法与技术路线 / 创新点与可行性 / 进度安排与预期成果。
2. 问题驱动六板块：研究背景与现实问题 / 现有研究与关键不足 / 科学问题与研究目标 / 研究思路、内容与方法 / 前期基础与创新点 / 计划、风险与预期成果。
3. 精简五板块：研究背景与意义 / 研究现状与研究空白 / 研究内容与技术路线 / 已完成工作与可行性 / 进度安排与预期成果。

允许展示已完成的方法设计、数据收集和预实验，但必须标注证据状态。不得把预期结果写成已验证成果。

## 中期考核

目标：证明研究按计划推进，已形成足够工作量和阶段结果，问题可控，剩余工作可按期完成。

优先分块：研究任务与原计划 / 已完成工作 / 阶段性成果 / 问题与调整 / 下一步计划。

方法介绍只保留理解阶段结果所必需的部分。完成比例必须来自任务清单或用户提供的学校要求，不能默认写成 70%。建议用“任务—状态—证据—偏差—剩余工作”对照表。

## 毕业答辩

目标：形成问题、方法、实验、结论、贡献和边界的完整闭环。

可选分块：

1. 经典五板块：研究背景与问题 / 研究内容与方法 / 实验与结果 / 创新与贡献 / 总结与展望。
2. 方法结果六板块：研究背景 / 现状与空白 / 研究目标与总体方案 / 关键方法 / 实验结果与讨论 / 贡献、局限与展望。

实验页必须说明比较对象、指标、数据集或样本、结果和解释。贡献要由前文证据支持；局限不能只写通用套话。

## 组会：文献精读

目标：讲清论文解决什么问题、如何解决、证据是否支持、哪些内容可用于本人课题以及下一步做什么。

默认分块：论文与作者信息 / 研究问题与动机 / 核心方法 / 实验与结果 / 创新与局限 / 课题启发与下一步。

基本信息可包含期刊或会议、年份、分区、影响因子、作者团队和研究领域。分区与影响因子属于时效信息，必须使用用户材料或经过验证的当前来源，并注明年份和口径。

方法图和结果图必须回答“图中比较什么、读出的结论是什么、为什么支持或限制作者主张”。单图页默认采用图文版式；只有多图对照或必须全幅辨认的高价值图才可主要展示图片。

课题启发不能停留在“值得借鉴”。至少写明可迁移机制、需要验证的问题、复现/消融组、评价指标和下一步动作。普通文献分享以核心判断和行动收束，不使用课堂式“请讨论”结尾，除非用户明确要求讨论议程。

## 组会：周报进展

目标：高效呈现本周任务、文献与学习、实验、失败尝试、问题和下周动作。

默认分块：本周任务 / 文献与学习 / 实验与结果 / 问题与讨论 / 下周计划。

允许记录软件学习、方法复现和必要的生活或状态事项，但后者是用户可选模块，不作为所有组会的必备内容。

## 组会：课题或论文进展

目标：围绕当前研究阶段展示已完成模块、阶段证据、缺失实验、写作状态和需要讨论的决策。

默认分块：课题目标与当前阶段 / 已完成研究内容 / 阶段结果 / 缺口与问题 / 下一步实验与写作。

不要做成缩短版毕业答辩。背景只保留一页以内，把时间集中在新证据、失败、缺口和下一步。

## 科研项目申报

目标：证明项目必要、目标可考核、路线可执行、团队有基础、风险可控。

默认分块：立项背景与必要性 / 目标与考核指标 / 研究内容与技术路线 / 创新点 / 基础与团队 / 计划、风险与产出。

## 科研项目比赛

目标：在较短时间内证明问题真实、方案有优势、验证充分且具有应用价值。

默认分块：问题与应用场景 / 解决方案 / 核心技术与创新 / 验证结果 / 应用价值与优势 / 团队与推进计划。

科研比赛允许更强的叙事节奏，但不得虚构市场、用户、实验或应用数据。

## 项目中期与结题

目标：按任务书审计目标、指标、成果、问题和交付状态。

默认分块：任务书与目标 / 任务完成情况 / 指标与成果证据 / 应用或影响 / 问题与整改 / 后续工作。

结题时用“约定指标—实际完成—证据—差异说明”逐项对照，不能用活动数量代替成果。

## 学术会议报告

目标：在会议时限内让同行快速理解研究问题、核心方法、最强结果和贡献边界。

默认分块：研究动机 / 问题与核心思路 / 方法 / 关键结果 / 贡献与结论。

优先保留最强证据，删除论文中不影响主张的细节。会议报告不是把完整论文压缩到小字号。

## 时间与页数

根据场景、时长和页面复杂度分配，不使用固定“一分钟一页”。每页必须记录 `time_seconds`，每个板块必须有时间和页数预算。开题重在必要性与可行性，中期重在已完成工作与阶段证据，毕业答辩重在方法—实验—贡献闭环，组会重在最新信息和下一步行动。
